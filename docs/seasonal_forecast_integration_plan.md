# 1か月予報を活用した長期水位予測（28日・60日・90日先）の精度向上案

## 1. はじめに
現状のAI予測システムでは、長期（28日後以降）の予測において「直近7日間の週間予報」をベースにした特徴量を使用しており、期間が長くなるほど予報の有効性が薄れる課題があります。
気象庁が提供する「1か月予報（アンサンブル予報に基づく確率予報）」をシステムに組み込むことで、この課題を解決し、長期予測の信頼性を向上させる手法を提案します。

---

## 2. 実装の実現可能性（Feasibility）

### データ取得の自動化
気象庁の防災情報システムが提供するJSON API（非公式ながら広く利用されているもの）から、東北地方の1か月予報データを直接取得可能です。
- **エンドポイント**: `https://www.jma.go.jp/bosai/season/data/P1M/010200.json` (010200 = 東北地方)
- **更新頻度**: 毎週木曜日 14:30
- **取得可能なデータ**: 向こう1か月の「気温」「降水量」「日照時間」の階級別確率（低い・平年並・高い）

### 実装手順
1. **Scraperの新規作成**: `MonthlyForecastScraper.js` を作成し、上記JSONから東北地方の確率値を定期取得する。
2. **変換ロジックの実装**: 確率値（例：少ない40%, 平年並40%, 多い20%）を、AIが理解可能な数値（期待値係数）に変換する。
3. **推論パイプラインへの統合**: `DataAggregator.js` で取得した係数を特徴量としてPythonモデルに渡す。

---

## 3. 数値化アルゴリズムの提案

確率予報は「階級（カテゴリ）」で提供されるため、これを**「平年比期待値（Expected Anomaly Index）」**に変換して入力します。

### 計算式例（降水量の例）
予報が「少ない: $P_{low}$」「平年並: $P_{normal}$」「多い: $P_{high}$」の場合：
$$\text{Expected precip index} = (0.7 \times P_{low}) + (1.0 \times P_{normal}) + (1.3 \times P_{high})$$
*※係数（0.7, 1.3）は過去の統計データに基づき最適化。*

この「期待値係数」を、その時期の**「平年値（CLIMATOLOGY）」**に乗じることで、向こう30日間の予測降水量合計を算出します。
- **入力特徴量**: `Forecast_Anomaly_30d_Precip`, `Forecast_Anomaly_30d_Temp`

---

## 4. 期待される精度向上度合

### 1. 「平年並」からの脱却
現在のモデルは、長期になるほど「月（Month）」という静的な特徴量に依存しがち（＝例年通りの動きを予測）です。1か月予報を導入することで、「今年の3月は例年より特に乾燥する」といった**「その年特有の傾向」**をモデルが捉えられるようになります。

### 2. 予測期間ごとの改善見込み
| 予測期間 | 改善期待度 | 理由 |
| :--- | :--- | :--- |
| **7日後** | 低〜中 | 週間予報（決定論的予報）の精度が高いため、1か月予報の影響は限定的。 |
| **28日後** | **高** | 週間予報が全く当たらない領域で、大気の大規模な流れ（1か月予報）が唯一の指針となるため。 |
| **60-90日後** | 中 | 1か月予報の先（2〜3か月予報）も存在し、それらを組み合わせることで「渇水の長期化」を予測可能。 |

### 3. 具体的なメリット
- **渇水リスクの早期検知**: 「降水量：少ない」の確率が高い場合、モデルがそれを検知して水位低下の角度を急激に予測するよう学習されます。
- **融雪時期の補正**: 気温の1か月予報が高い場合、融雪による流入増が例年より早まることを予測に反映できます。

---

## 5. 結論と次のステップ
気象庁のJSONデータを活用した実装は**技術的に十分可能**であり、特に28日後以降の長期予測においては、現状の「週間予報の単純延長」よりも**有意な精度向上**が見込まれます。

**推奨されるアクション**:
1. `MonthlyForecastScraper.js` の試作（APIフェッチのみ）。
2. 過去の1か月予報データを用いた学習用データセットの再構築（バックテスト）。
3. ランダムフォレストモデルへの特徴量追加と再学習。
