# 鳴子ダム 渇水予測モデル構築のご提案

ご提供いただいた `dum_data` フォルダ内のデータ（鳴子ダムの過去データ、および周辺気象観測所のデータ）を確認いたしました。
これらのデータを組み合わせることで、**「現在の気象データや水位から、数週間〜数ヶ月先の鳴子ダムの貯水位（渇水危険度）を予測する」** 手法の構築は十分に**可能**です。

以下に、分析・予測アプローチの手順と手法をご提案いたします。

---

## 1. 取得済みデータの整理と前処理（データラングリング）
提供データは形式がバラバラなため、まずはコンピューターが学習しやすい形（1つの連続したテーブル）に結合・整形する必要があります。

*   **鳴子ダムデータ (`1993_2023_day_storage_inflow_discharge (2).csv`)**
    *   期間：1993年〜2023年
    *   目的変数（予測したい値）：**貯水位**
    *   特徴量：流入量、放流量
*   **気象データ (`hurukawa`, `kawatabi`, `komanoyu`, `nikkawa`, `shiroishi`)**
    *   期間：2000年〜（一部異なる可能性あり）
    *   特徴量：降水量、降雪・積雪深、最高/平均/最低気温
*   **前処理のステップ**:
    1.  すべての日付フォーマットを統一（YYYY/MM/DD → `YYYY-MM-DD`）
    2.  2000年〜2023年の**期間をクロスジョイン（結合）**し、1日1行のデータフレームを作成
    3.  欠損値（NaN）の補完（線形補間や直近値の代入など）

## 2. 予測アプローチ（モデリング手法）の提案
予測の要件（何日先を予測したいか、精度と解釈性のどちらを重んじるか）によって最適なアルゴリズムが異なります。

### 手法A: 回帰分析による貯水位の直接予測 (LightGBM / XGBoost)
現在の水位と過去数日間の気象データを入力し、「N日後の貯水位」を直接出力させる手法です。
*   **特徴量エンジニアリング**: 
    *   過去7日間、30日間の「累積降水量」「平均気温」などの移動平均を計算して入力に加えます。
    *   「春の雪解け」パラメータとして、過去の累積積雪深と直近の気温の掛け合わせ指標を作成します（鳴子ダムのような多雪地帯では最重要）。
*   **メリット**: 実装が比較的容易で、どの気象要因が水位低下に効いているか（Feature Importance）を解釈しやすい。
*   **デメリット**: 連続した時系列の「波」を捉えるのは少し苦手。

### 手法B: 時系列ディープラーニングモデル (LSTM / Prophet / TimeSformer)
時間の流れ（トレンド）と季節性（毎年起こるパターン）を学習させる手法です。
*   **FB Prophet**: 毎年の渇水期・豊水期のトレンドを自動抽出するアルゴリズム。「例年に比べて今年の降水量がこれくらい少ないから、来月の水位はこのくらい下がるだろう」という予測が得意。
*   **LSTM / GRU (RNN系)**: 過去のシーケンスデータ（例：過去60日分の水位・気温・降水）をそのまま入力し、未来のシーケンスを出力する。
*   **メリット**: 長期的な予測（1〜2ヶ月先）のトレンドを捉えるのに適している。
*   **デメリット**: 突発的なゲリラ豪雨などの表現が難しい。

## 3. 「渇水危険度」への判定ロジック
モデルが「30日後の水位は〇〇mになる」と予測した値に対して、既存アプリに組み込んでいる「渇水レベル判定（正常、注意、警戒、危険）」の閾値を当てはめます。
*   例: *「現在の水位は正常ですが、機械学習の予測では30日後に『警戒レベル』に突入する確率が75%です」* というアラートをダッシュボードに出す。

## 4. システムへの組み込み（実装ロードマップ）
もしこのシステムを現在のNode.jsアプリに組み込む場合、以下のステップになります。

1.  **Python環境の構築**: データ分析と機械学習はNode.jsよりもPython（Pandas, Scikit-learn等）が圧倒的に適しています。
2.  **学習スクリプトの作成**: `dum_data` を読み込んで学習済みのモデルファイル（`.pkl` など）を生成するPythonスクリプトを作成。
3.  **推論APIの作成**: Pythonで軽量な推論用API（FastAPI等）を立ち上げ、Node.jsバックエンド（現在の `DataAggregator.js`）から「現在の水位と気象」を送信して、Python側から「予測水位」を返してもらう構成にします。

いかがでしょうか？
もしよろしければ、まずはPythonを使って**提供データを結合し、基礎分析（どの気象データと水位の相関が強いか等）を行うスクリプト**の作成から着手することが可能です。
